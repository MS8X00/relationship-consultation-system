# Security Policy

## Reporting a Vulnerability

If you find a security issue, do not open a public issue first.

Please report privately via email:

- security contact: `gbanyan@users.noreply.github.com`

Include:

- affected file(s) and path(s)
- reproduction steps
- potential impact
- suggested mitigation (if available)

## Response Targets

- initial acknowledgment: within 72 hours
- triage update: within 7 days

## Scope Notes

This repository is primarily prompt/skill content. Security reports are most useful when they involve:

- prompt-injection amplification patterns
- unsafe escalation patterns
- harmful instruction leakage or bypass behavior
